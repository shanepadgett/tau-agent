- When the user runs `/manage-sessions` in TUI mode while the agent is idle, Tau shall open a session management panel.
- When the user runs `/manage-sessions` outside TUI mode, Tau shall report that the session manager requires TUI mode.
- When the session management panel opens, Tau shall show active sessions for the current folder by default.
- When the session management panel shows active sessions, Tau shall omit the currently active session.
- When the session management panel shows archived sessions, Tau shall list archived sessions from the archive root.
- When the user presses `s` in the session management panel, Tau shall toggle between current-folder scope and all-projects scope.
- When the session management panel changes scope, Tau shall clear existing selections.
- When the user switches tabs in the session management panel, Tau shall switch between active sessions and archived sessions without combining tab behavior with list selection behavior.
- When Tau renders a session row, Tau shall show the session display name and how long ago it was last updated.
- When a session has a user-defined name, Tau shall use that name as the row display name.
- When a session has no user-defined name, Tau shall use the session first message as the row display name.
- When the user presses Space on a session row, Tau shall toggle that row selection.
- When the user presses `c` in the session management panel, Tau shall clear the current tab selection.
- When the current tab has selected rows and the user starts an action that supports selections, Tau shall target the selected rows.
- When the current tab has no selected rows and the user starts an action that supports a single row, Tau shall target the highlighted row.
- When the active sessions tab is visible and the user presses `a`, Tau shall prepare to archive the selected or highlighted active sessions.
- When the active sessions tab is visible and the user presses Shift+A, Tau shall prepare to archive every active session strictly older than the highlighted row.
- When the active sessions tab is visible and the user presses `d`, Tau shall prepare to delete the selected or highlighted active sessions.
- When the active sessions tab is visible and the user presses Shift+D, Tau shall prepare to delete every active session strictly older than the highlighted row.
- When the archive tab is visible and the user presses `u`, Tau shall prepare to unarchive the selected or highlighted archived sessions.
- When the archive tab is visible and the user presses Shift+U, Tau shall prepare to unarchive every archived session strictly older than the highlighted row.
- When the archive tab is visible and the user presses `d`, Tau shall prepare to delete the selected or highlighted archived sessions.
- When the archive tab is visible and the user presses Shift+D, Tau shall prepare to delete every archived session strictly older than the highlighted row.
- When Tau prepares a destructive or file-moving bulk action from the session management panel, Tau shall show an inline footer acknowledgement with confirm and cancel key hints.
- While an inline footer acknowledgement is pending, Tau shall ignore keys other than confirm and cancel.
- When the user confirms a pending archive action, Tau shall move each targeted session file from Pi's sessions root to the mirrored path under the archive root.
- When the user confirms a pending unarchive action, Tau shall move each targeted session file from the archive root back to the mirrored path under Pi's sessions root.
- When the user confirms a pending delete action, Tau shall use the trash CLI when it succeeds and shall permanently delete the session file when trash is unavailable or fails.
- When a session action completes in the panel, Tau shall refresh the visible session lists and report failures without closing the panel.
- When the user presses Escape in the session management panel with no pending acknowledgement, Tau shall close the panel.
- When the user presses Enter in the session management panel with no pending acknowledgement, Tau shall not resume or open a session.
- When the user runs `/sweep`, Tau shall show a native Pi selection UI with archive and delete choices for the current session.
- When the user chooses a `/sweep` action, Tau shall ask for confirmation with Pi's native confirmation UI.
- When the user confirms `/sweep`, Tau shall switch to a blank new session before mutating the old current session file.
- When the user confirms archive in `/sweep`, Tau shall move the old current session file to the archive root after the new session is active.
- When the user confirms delete in `/sweep`, Tau shall delete the old current session file after the new session is active.
- When the user cancels `/sweep`, Tau shall not switch sessions and shall not mutate the current session file.
- If the current session is not persisted, Tau shall refuse `/sweep`.
- When Tau archives sessions, Tau shall store archived files under `join(getAgentDir(), "session-archive")` and shall preserve Pi's encoded project folder and session filename.
- When Tau unarchives sessions, Tau shall restore archived files to the matching relative path under `join(getAgentDir(), "sessions")`.
- When Tau creates shared TUI components for this feature, Tau shall keep panel shell, tabs, and multiselect list behavior in separate composable components.
- When a shared TUI component provides default key hints, Tau shall expose those hints so a parent can pass them to the panel footer.
- When Tau adds shared TUI components for this feature, Tau shall document how to compose them in `docs/`.
